# Workflow SDK

Bind Hitl to [Vercel Workflow SDK](https://workflow-sdk.dev). This page covers server setup with `workflowResolver` and the workflow-side client that suspends over a durable `"use step"` fetch.

Vercel Workflow SDK runs workflows in a separate sandbox from your app server. The workflow client carries no state backend. It suspends and POSTs to the server over a durable `"use step"` fetch.

If you are following the [Quickstart](/docs/quickstart), this page covers the server and workflow client setup.

## Install

```bash
npm i @hitl-sdk/hitl @hitl-sdk/resolver-workflow-sdk workflow
```

## Set up the Hitl server

```typescript
// lib/hitl.ts
import { Hitl } from "@hitl-sdk/hitl";
import { workflowResolver } from "@hitl-sdk/resolver-workflow-sdk";

export const hitl = new Hitl({
  state, // see State docs
  resolver: workflowResolver(),
});
```

Mount `hitl.routeHandlers` at `/.well-known/hitl/v1`. See [Host integration](/docs/host-integration).

## Create the workflow client

```typescript
// lib/hitl-client.ts
import type { HitlRequest } from "@hitl-sdk/hitl/core";
import { createWorkflowSdkHitlClient } from "@hitl-sdk/resolver-workflow-sdk";

async function hitlRequest(req: HitlRequest) {
  "use step";
  const res = await fetch(req.url, {
    method: req.method,
    headers: req.headers,
    body: req.body,
  });
  return { status: res.status, ok: res.ok, body: await res.text() };
}

export const { waitForHuman, requestHuman, notify } = createWorkflowSdkHitlClient({
  request: hitlRequest,
});
```

Import `waitForHuman` from this module in workflow functions, not from `@hitl-sdk/hitl` directly.

## Approval flow

1. Workflow calls `waitForHuman`, suspends, and POSTs to the server.
2. Server persists the request, delivers to inbox/adapters, and waits.
3. Reviewer approves. `workflowResolver` resumes the workflow with the result.

If you use Vercel Workflow SDK, configure its [world](https://workflow-sdk.dev/worlds) separately. Hitl `state` is independent; see [State docs](/docs/state/sqlite).
