# waitForHuman

The core primitive. Suspend a workflow until a human approves, denies, or edits fields.

## Basic usage

```typescript
import { actions, field, isResolved } from "@hitl-sdk/hitl";
import { waitForHuman } from "@/lib/hitl-client";

const approval = await waitForHuman({
  message: "Send this email?",
  actions: actions()
    .approve({
      label: "Send",
      fields: {
        subject: field.textField({ label: "Subject", default: draft.subject }),
        body: field.textArea({ label: "Body", default: draft.body }),
      },
    })
    .deny({
      label: "Reject",
      fields: { reason: field.textArea({ label: "Reason" }) },
    })
    .build(),
  timeout: "72h",
});

if (!isResolved(approval, "approve")) {
  return; // denied, timed out, or cancelled
}

const { subject, body } = approval.feedbacks; // typed for the approve action
```

## requestHuman + waitForHuman

Split creation and waiting when you need to notify between them:

```typescript
const pending = await requestHuman({ message: "...", actions });

await notify({ after: pending, message: "Context for reviewer" });

const result = await waitForHuman(pending, {
  reminders: [remind.after("1h", { message: "Still waiting" })],
});
```

## Actions and fields

`actions()` builds approve/deny/custom buttons. Attach `field.*` helpers for editable fields on each action.

`isResolved(result, "approve")` narrows the result and types `feedbacks` to the chosen action's fields.

## Timeout

Pass `timeout: "72h"` (or ms number). On timeout, the server calls the resolver with a timeout payload; `isResolved` returns false.

## Reminders and escalation

```typescript
import { remind, escalate } from "@hitl-sdk/hitl";

await waitForHuman(pending, {
  reminders: [
    remind.after("1h", { message: "Still waiting" }),
    escalate.to("approvals:slack:C999").after("4h", { mode: "redeliver" }),
  ],
});
```

## Batch approvals

For sequential multi-reviewer flows, use batch APIs (`requestHuman` batch mode, `resolveBatch`). See package types and tests in the repo.

## Import from the workflow client

Always import `waitForHuman` from your engine-bound client module (`lib/hitl-client.ts`) — not from `@hitl-sdk/hitl` inside `"use workflow"` functions.
