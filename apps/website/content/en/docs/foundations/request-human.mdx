# requestHuman

The create phase of a human step. Delivers the approval to a channel and returns a pending handle without suspending the workflow yet.

## Import

```typescript
import { requestHuman } from "@/lib/hitl-client";
```

## Signatures

```typescript
// Single request
requestHuman(opts: RequestHumanOptions<Actions>): Promise<HumanPending<Actions>>

// Batch request
requestHuman(opts: RequestHumanOptions<Actions> & { items: HumanItem[] }): Promise<HumanBatchPending<Actions>>
```

## Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `message` | `string` | Yes* | Prompt shown to the reviewer |
| `actions` | `HumanActions` | Yes | Built with [`actions()`](/docs/foundations/actions-and-fields) |
| `context` | `Record<string, unknown>` | No | Opaque metadata stored with the request |
| `channel` | `string` | No | Adapter id or `adapter_id:destination`; defaults to first configured adapter |
| `after` | `HumanResult \| TimelineAnchor` | No | Post under the same chat thread as a prior step or [`notify`](/docs/foundations/notify) |
| `items` | `HumanItem[]` | No | Batch mode: multiple decisions in one delivery |
| `defaultsActionId` | `string` | No | Which action receives per-item `defaults` when no approve action exists |

\*Required when `items` is absent.

### `HumanItem`

| Field | Type | Description |
|-------|------|-------------|
| `message` | `string` | Per-item prompt in batch mode |
| `defaults` | `Partial<FeedbackValues>` | Override submit field defaults for this item |

## Returns

| Type | Description |
|------|-------------|
| `HumanPending<Actions>` | Single request; pass to [`waitForHuman`](/docs/foundations/wait-for-human) or [`notify`](/docs/foundations/notify) |
| `HumanBatchPending<Actions>` | Batch request; `id` is the batch id |

Both implement `TimelineAnchor` (`id`, `externalRef`) for thread chaining.

## Example

```typescript
import { actions, field } from "@hitl-sdk/hitl";
import { requestHuman, waitForHuman, notify } from "@/lib/hitl-client";

const pending = await requestHuman({
  message: "Approve deployment to production?",
  actions: actions()
    .approve({ label: "Ship it" })
    .deny({ fields: { reason: field.textArea({ label: "Reason" }) } })
    .build(),
});

await notify({
  after: pending,
  message: "Diff: https://github.com/org/repo/compare/main...release",
  detail: { commit: "abc123" },
});

const result = await waitForHuman(pending, { timeout: "24h" });
```

## Thread chaining with `after`

Post a follow-up request in the same chat thread:

```typescript
const first = await waitForHuman({ message: "Initial review?", actions });

const second = await requestHuman({
  message: "Follow-up: any changes needed?",
  actions,
  after: first, // same thread as the resolved first step
});

const followUp = await waitForHuman(second);
```

## Batch mode

```typescript
const batch = await requestHuman({
  actions,
  items: [
    { message: "Approve line item 1" },
    { message: "Approve line item 2", defaults: { amount: "500" } },
  ],
});

const results = await waitForHuman(batch, { timeout: "48h" });
```

Batch items require an approve action (or explicit `defaultsActionId`) for per-item defaults.

## When to use

| Pattern | Use when |
